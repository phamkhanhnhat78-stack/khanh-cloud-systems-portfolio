# CloudOps Incident Lab

CloudOps Incident Lab is a standalone static project for cloud and IT support troubleshooting practice.

It is intentionally not a portfolio. The personal portfolio lives in:

```text
https://github.com/phamkhanhnhat78-stack/phamkhanhnhat78-stack.github.io
```

This project is a technical lab that demonstrates how operations thinking can be translated into cloud incident response.

## Project Purpose

The lab simulates common cloud support incidents:

- Public application unreachable
- Wrong DNS or public endpoint
- Backend target unhealthy
- Storage access risk
- Monitoring or alerting gap

Each incident includes:

1. User impact.
2. Affected cloud layer.
3. Likely root cause.
4. First response checklist.

## Skills Demonstrated

- Incident triage
- User-impact analysis
- AWS support fundamentals
- Route table and Internet Gateway checks
- DNS and edge endpoint validation
- Load balancer and EC2 target health checks
- S3 and IAM access review
- CloudWatch and SNS alerting logic
- Runbook-style documentation
